# Sentiment Analysis: From Rules to Machine Learning

## 📌 Overview

This repository demonstrates the evolution of a sentiment analysis system through three different approaches.

Rather than directly using Machine Learning, the project begins with a simple rule-based implementation, progresses to the VADER sentiment analyzer, and finally moves to a Machine Learning model.

The objective is to understand how sentiment analysis techniques improve in accuracy and capability as more advanced approaches are introduced!
 
---

## 🚀 Project Stages

### 📁 Rule_Based

A sentiment analyzer built entirely using Python logic.

Features:

* Keyword matching
* Positive and negative word dictionaries
* Negation handling (`not good`)
* Punctuation removal
* Sentiment classification

---

### 📁 VADER

A lexicon and rule-based sentiment analyzer.

Features:

* VADER Sentiment Analyzer
* Compound sentiment scoring
* JSON report generation
* Summary statistics
* Better handling of punctuation, capitalization and intensity

---

### 📁 Machine_Learning

A supervised learning approach.

Planned Features:

* Dataset preprocessing
* Feature extraction
* Logistic Regression
* Model evaluation
* Prediction on unseen reviews
