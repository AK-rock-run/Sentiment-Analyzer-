# Rule-Based Sentiment Analysis

## 📌 Description

This project is the first stage of the Sentiment Analysis journey.

The sentiment of a review is determined using manually created rules and sentiment dictionaries.

No Machine Learning libraries are used.

---

## Features

* User-defined positive and negative word lists
* Text preprocessing
* Case normalization
* Punctuation removal
* Negation handling
* Positive, Negative and Neutral classification

---

## Workflow

Input Review

↓

Preprocessing

↓

Keyword Matching

↓

Sentiment Classification

↓

Display Result

---

## Technologies

* Python

---

## Advantages

* Easy to understand
* Fast execution
* No training required

---

## Limitations

* Limited vocabulary
* Cannot understand context
* Poor handling of complex sentences
* Requires manual rule creation

---

## Future Scope

The next stage improves these limitations using the VADER sentiment analyzer.
